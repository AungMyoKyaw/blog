Learning How to Learn [First Week]

ဘာတွေ ရေးလို့ ရေးထားလဲ မသိလို့ ပြန်ဖြုတ်ထား တဲ့ ဟာ ကို repost လိုက်တယ်။ နည်းနည်းတော့ ကြာနေပြီ ဟိုးအရင်က medium မှာ ထားပြီးတော့ unpublish ထားတာ ။ အချို့ စာကြောင်းတွေ က ဆောင့်ကြွားကြွား ဆိုတော့ သည်းခံ ဖတ် ကြပါ။ 🤣🤣

တခါလာလည်း technical တခါလာလည်း technical နဲ့ကြာလာတဲ့ အခါလူလည်း ထုံပေပေဖြစ်လာတော့ course တက်ဖို့ အရမ်းပျင်းလာတယ်။ မလုပ်ပဲနဲ့ ထိုင်နေလို့ မရလို့သာ ဆက်လေ့လာဖြစ်တယ် ကြာလာတော့လည်း ပျင်းလာတယ်။ အပြောင်းအလဲ ဖြစ်အောင် လို့ Learning How to Learn ဆိုတဲ့ course တစ်ခု တက်ဖြစ်တယ်။

ဒီ course ကို recommend ပေးထားတဲ့ သူတွေလည်း များတယ်။ ကိုယ် enroll လုပ်ထားတာကို နှစ်ပေါက်နေပြီ။ အခု နောက် ဆုံး ကြံရာမရဖြစ်နေတဲ့ အချိန်မှာမှပဲ တက်ဖြစ်တော့တယ်။

အခု လောလောဆယ် week 1 ပဲ ရောက်သေးတယ်။ ဒါပေမယ့် week 1 မှာပြောသွားတာလေးတွေနဲ့ကိုပဲ motivation ပြန်တက်လာတော့ ဆုံးအောင် တက်ရမယ့် course တစ်ခု လို့ပြောလို့ရတယ်။

ပထမဆုံး သူပြောသွားတာက လူ့ ဦးဏှောက် ရဲ့ mode နှစ်ခု အကြောင်းပဲ။

focused mode ရယ် diffuse mode ရယ်။

focused mode ကတော့ ကျနော်တို့ ခဏခဏ သုံးနေကြပဲ ။ ဥပမာ — တစ်ခုခုကို စဉ်းစားတယ် ဆိုရင် အာရုံ စိုက်ပြီးတော့ မရရအောင် စဉ်းစားတယ်။ အဲ့တာ focused mode ပဲ။

diffuse mode ကိုတော့ အာရုံပျံလွှင့်တဲ့ mode လို့ပြောလို့ရမယ်ထင်တယ်။ ဒါပေမယ့် အထင်သေးလို့မရဘူး။ diffuse mode မှာ လျှပ်တပြတ်ဝင်လာတဲ့ အတွေး တွေက focused mode မှာ လုံးဝ စဉ်းစား လို့မရတာတွေ ရှိတယ်။ ဥပမာ Archimedes က ရေချိုး နေရင်း သူစဉ်းစားနေတဲ့ ပြဿနာ ရဲ့ အဖြ ကို ရှာတွေ့သွားသလိုမျိုးပေါ့ [Archimedes’_principle]

ဘယ် mode က ကောင်းတယ် လို့ ဇမ်းတင်ပြီးတော့ ပြောတာတော့မဟုတ်ဘူး ရယ်။ တခုမဟုတ်တခု ကတော့ အသုံးဝင်တာချည်းပဲ ကိုယ် လိုသလို သုံးသွားရမှာပဲ။

ဥပမာ — အတွေးသစ် အမြင်သစ်တွေလိုချင်တဲ့ အချိန်မှာ diffuse mode ကို သွားပြီးတော့ implement လုပ်တဲ့ အချိန်ကြတော့ focused mode ကို သွားသလိုမျိုးပဲပေါ့။

ဘာကြောင့် diffuse mode က စဉ်းစားဉာဏ် ပိုကွန့်မြူးနိုင်လဲ ဆိုတာက ဦးဏှောက်ရဲ့ ဖွဲ့စည်းပုံနဲ့ ဆိုင်တယ်။ လူ့ဦးဏှောက်မှာ က neuron တွေနဲ့ တစ်ခုနဲ့ တစ်ခု ချိတ်ဆက် အလုပ်လုပ်တယ်။ focused mode မှာ neuron တွေက တစ်ခုနဲ့ တစ်ခု ပူးပူးကပ်ကပ် အလုပ်လုပ်တယ်။ diffuse mode မှာကြတော့ ခပ်ကွာကွာ အလုပ်လုပ်တယ်။ ဒီ course မှာတော့ pinball game နဲ့ ဥပမာ ပြထားတယ်။ တခုနဲ့ တခု ကွာဝေးတဲ့ အခါမှာ focus mode မှာ ချိတ် နေကြ neuron တွေ နဲ့ မတူပဲ လွတ်လွတ်လပ်လပ် ရွေ့ နိုင်တဲ့ အတွက် စိတ်ကူးသစ်တွေ ပိုရဖို့ အတွက် အခွင့် အရေးဖြစ်နေပါတယ် တဲ့။

diffuse mode ကို ဘယ်လို ရောက်နိုင်လဲဆိုတော့ လူမှာ သိစိတ် နဲ့ မသိစိတ် နှစ်ခု ရှိတဲ့ ထဲ့မှာ မသိစိတ် တဝက် သိစိတ်တဝက် ကြားကာလလေးမှာ ရောက်သွားပါတယ်တဲ့ ။[ ဘုရား ဟော အရတော့ စိတ် ဆိုတာ တစ်ခု ပါရှိပါတယ် ] ။ သိစိတ် တဝက် မသိစိတ် တဝက် ကာလာကို ရောက်သွားအောင် Edison သုံးတဲ့ နည်း ကတော့ ဘောလုံး လေး ကိုင်ပြီးတော့ မှိန်း နေ လိုက်တာပါပဲ။။ အိပ်ချင်လာတဲ့အခါ ဘောလုံးပြုတ်ကျသွားတယ်။ အဲ့မှာ ဘောလုံးအသံကြောင့်တခါပြန်ပြီးတော့သတိဝင်လာတယ်။အဲ့လို အိပ်မှုန်စုံဝါး အချိန်က အတွေးသစ်တွေက လျှပ်စီးလက်သလိုဝင်လာတက်ပါတယ်တဲ့။ အဲ့တာတွေကို လက်တွေ အကောင်အထည်ဖော်လိုက်တာပါပဲ။

လူ့ဦးဏှောက်ဆိုတာ neuron တွေများလာနဲ့ အမျှ စွမ်းဆောင်ရည် တက်လာတယ် ဆိုတော့ neuron တွေ များလာဖို့ အတွက်ဘယ်လိုလုပ်မလဲ ဆိုတဲ့ မေးခွန်းရှိပါတယ်။ သိပ္ပံ ဆိုင်ရာ လက်တွေ့ စမ်းသပ်ချက် တစ်ခုကို စမ်းကြည့်ပါတယ်။ ကြွက်တွေနဲ့ပေါ့။ အဲ့မှာ ဆိုရင် community နဲ့ နေတဲ့ ကြွက်ရယ် မနေတဲ့ ကြွက်ရယ် ခွဲကြည့်တဲ့အခါမှာ community မှာ လှုပ်ရှားမှုများတဲ့ ကြွက်က ပိုပြီးတော့ neuron အရေအတွက် များလာတာကို တွေ့ရပါတယ်။ အဲ့တာ community မှာ အသက် ဝင် ဖို့ လိုပါတယ်။ ကိုယ့် ပတ်ဝန်းကျင်က creative ဖြစ်တဲ့ သူတွေနဲ့ ဝန်းရံ နေတဲ့ အခါ ကိုယ်ပါ creative ဖြစ်လာပါတယ်။

အဲ့ လိုမျိုး အခြေအနေမျိုး မရနိုင်ရင်တော့ လေ့ကျင့်ခန်း လုပ်တာဟာ neuron များလာဖို့ အထောက်အကူဖြစ်ပါတယ်။ အဲ့တော့ ရုပ်ပိုင်း ဆိုင်ရာ လေ့ကျင့်ခန်းတွေက ဦးဏှောက်စွမ်းရည် တိုးတက်ဖို့အတွက် အရေးပါပါတယ်။(ဒီမှာတော့ ကစားချိန် မပေးပဲ စာဖင်ထောင် ကျက်ခိုင်းနေတာပါပဲ)။

အိပ်တယ် ဆိုတာကလည်း ဦးဏှောက် ကျန်းမာရေး မှာ အထောက်အကူပြုတယ်။ လူဦးဏှောက်မှာ အချိန်ကြာလာတာနဲ့ အမျှ Neuron လေးတွေကြားမှာ အဆိပ်အတောက် တွေ ပေါ်လာတက်တယ်။ သူ့ကို ရှင်းပေးဖို့ ဆိုရင် ရှိသင့်တဲ့ အိပ်ချိန် တစ်ခု တော့ ပေးဖို့ လိုအပ်တယ်။

မှတ်ဉာဏ် နဲ့ ပတ်သက်ပြီးတော့ ရှင်းပြထားတာ စိတ်ဝင်စားဖို့ ကောင်းတယ်။ လူမှာ ရေရှည်မှတ်ဉာဏ်(long term memory) ရေတို မှတ်ဉာဏ် (short term memory)ရယ် ဆိုပြီးတော့ ရှိတယ်။ short term memory ထဲမှာ ထားတာတွေက ခန္တာကိုယ် ရဲ့ ဇီဝဖြစ်စဉ်ကြောင့် အချိန်နည်းနည်း ကြာတာနဲ့ မေ့သွားတက်တယ်။ အဲ့တော့ ကိုယ် အသုံးလိုတာတွေကို long term memory ထဲ့ကို ပို့ရတယ်။ ပို့တဲ့ နည်းက ဒီနေရာမှာ အရေးပါတယ်။ ကျနော်တို့ စာမေးပွဲ ဖြေမယ့် ညမှ စာကြည့်ရင် ဘယ်လောက် ကြည့်ကြည့် short term memory ထဲ့ ရောက်ဖို့ကများတယ်။ အဲ့ တော့ ဖြေခါနီး မနွှေးမိတာနဲ့ အကုန်မေ့ပြီးတော့ ခွေးဖြစ်တော့တာပါပဲ။ long term memory ထဲ ရောက်ဖို့ဆိုရင် အချိန်လိုပါတယ်။ အိမ်ဆောက်တဲ့အခါ ကျည်းခွေ ခွေပြီးတာနဲ့ အုတ်တွေ တစ်ခါထဲအပြီးအစီး တက်လို့မရပါဘူး အင်္ဂတေ ခြောက်ဖို့ အချိန်လို သလိုပဲ long term memory ထဲ ရောက်ဖို့ဆိုရင် ခဏ ခဏ ပြန်ခေါ် ရပါတယ်။ တခါတည်း စာကို တွန်းကျက် တာထက် ရလောက်တဲ့အချိန်အထိပဲ ကျက်ထားပြီးတော့ တနေ့ချင်းပြန်နွေးမယ် ဆိုရင်လူလည်း ပိုသက်သာပါတယ်။ အချိန်တန်ရင်ရမယ်ဆိုတဲ့ စိတ်ထားလိုက်တာ ပိုပြီးတော့ သက်တောင့်သက်တာ ရှိတယ်။

pomodoro ဆိုတဲ့ နည်းစနစ်ကို သုံးဖို့လည်း တိုက်တွန်းသွားတယ်။ သူက 25 မိနစ်လုပ်မယ်။ ၅ မိနစ် နားမယ် ဆိုတဲ့ နည်းစနစ်ပဲ။ လူတယောက် ဟာ အနည်း ဆုံး 25 မိနစ် လောက်ကတော့ ကောင်းကောင်းအာရုံ စိုက်နိုင်မှာပဲ။ ပြီးရင် 5 မိနစ် နား။ အလုပ် လုပ်နေရင်းနဲ့ အာရုံပျံလွှင့်တာကို ကောင်းကောင်း တာဆီးလို့ရမယ်။ လူလည်း သက်သာသွားမယ်။ ဒီ နည်းစနစ်ကို သုံးဖို့လိုတယ်။ ဘာလို့လည်း ဆိုတော့ အားအား နေ facebook ကို သွားပြီးတော့ ဖွင့်နေမိတဲ့ ကိစ္စက ဒီလိုမှပဲ အဆင်ပြေသွားမှာ ကိုး။

Interview လုပ်ထားတာတွေကို နားထောင်ရတာလည်း စိတ် ဝင်စားစရာကောင်းတယ်။

Benny နဲ့ အင်တာဗျုး မှာ လူတွေကို ဘာသာစကားသင်တဲ့ နေရာမှာ မှားနေတာ တွေကို ထောက်ပြသွားတာ သဘောကျစရာကောင်းတယ်။ ဒီဘာသာစကား တခု တက်ရင် လူရှိန်သွားမယ် ဂျွတ်သွားမယ် ဆိုတဲ့ စိတ်ကြီးနဲ့ သင်တာထက် တကယ့်ချစ်စိတ်နဲ့ သူတို့ လူမျိုး ရဲ့ စာပေ ၊ ယဉ်ကျေးမှု ဓလေ့ ထုံးတမ်း တွေကို ချစ်တဲ့ စိတ် သိချင်တဲ့ စိတ် လေ့လာချင်တဲ့ စိတ်နဲ့ လေ့လာတာ ပိုခရီး ရောက်တယ်တဲ့။( ဒါမျိုးက ဘာသာစကားတင်မဟုတ်ပါဘူး။ အခြား ဘာလေ့လာလေ့လာ ချစ်တဲ့ စိတ်လေး ရှိမှ အဆင်ပြေတယ်။ ပညာ ကို ခိုင်းစားဖို့ထက် ပညာကို ချစ်တဲ့ စိတ်နဲ့ သင်မှ ပညာ တတ်မှာ ။ နို့မို့ဆိုရင် တော့ တတ်လည်း ပုံမှန်ပဲ ဖြစ်မှာပဲ။ )

နောက် တစ်ခု က writing coach တစ်ယောက် ဖြစ်တဲ့ Daphne Gray-Grant နဲ့ အင်တာဗျူး။ စာရေးတဲ့ နေရာမှာ mode နှစ်ခု ရှိတယ်တဲ့ writing mode ရယ် editing mode ရယ် နှစ်ခုစလုံး ကို တပြိုင်နက် တည်း ဖြစ် ဖို့ ဆိုတာ မဖြစ်နိုင်ဘူး ။ ရေးနေတဲ့ အချိန် မှာ တည်းဖြတ်ဖို့ မစဉ်းစားနဲ့ ရေးမယ်သာရေး။ ငါရေးတာ ကောင်းပါ့ မလား ဘယ်လိုလေး ရေးလိုက်မယ် ဆိုတာထက် ကိုယ့် စိတ် လေးကို လွှတ်ပြီးတော့ diffuse mode နဲ့ ရေးရတယ်တဲ့။မကောင်းတော့လည်း မကောင်း စေချင်လို့ ရေးတာမှမဟုတ်ပဲ ကိုယ် နိုင်သလောက် ကြိုးစားရေးတာကို အပြစ်တင်ဖို့မကောင်းဘူး။ မကောင်းရင် မဖတ်ပဲ နေနိုင်တာပဲ။ အဲ့ ဒီ အချက်က ကျနော့ အတွက် အရမ်း အသုံးဝင်တယ်။ ကောင်းတာမကောင်းတာထက် ရေး ဖို့ ကအရေးကြီးတယ် ဘယ်သူမဆို အကောင်းဆုံး တွေ ဆိုတာ စစခြင်း တန်း ရေးနိုင်တာမှ မဟုတ်ပဲ။ လေ့ကျင့်ရင်းနဲ့ ကောင်းလာတာကို သတိထားသင့်တယ်။ မရေးတော့ ကောင်းတာလည်း မဖြစ် မကောင်းတာလည်း မဖြစ် ဘာအဖြစ်မှ မရှိတော့ ပိုဆိုးသွားရော။ အဲ့ လို ရေးပြီး ပြီ ဆိုတာနဲ့ editing mode ကို ပြန်သွား ကိုယ်ရေးတာ ကိုယ်ပြန်စစ် ပြန်တည်းဖြတ် ။ အဓိကက ရေးလိုက် တည်းဖြတ်လိုက်မလုပ်ဖို့ပဲ။ နို့မို့ဆို ခရီး မတွင်တော့ဘူး။ လက် ရှိ ကိုယ် ရေးနေတဲ့ စာလုံး တလုံးချင်းစီမှာပဲ အာရုံ ရှိရမယ်။

ဒီ course က တကယ်ကို အကျိုး ရှိတယ်။ programmer တွေ ဆိုတာမျိုးက အသစ် တွေအတွက် အမြဲတမ်း အသင့်ဖြစ်နေရမှာ ဆိုတော့ learning ဆိုတာ စဉ်ဆက်မပြတ်လုပ်နေရမယ့် ကိစ္စ ဖြစ်တော့ သင်ယူပုံသင်ယူနည်းကို သာ ကောင်းကောင်းသိမယ် ဆိုရင် တသက်လုံး အသုံးဝင်မယ့် skill ပဲ။ လောလော ဆယ်တော့ week 1 ပဲပြီးသေးတယ်။ week တခုချင်းစီအလိုက် blog post တစ်ခု ရေးမယ် စဉ်းစားတယ်။

Learning How to Learn [Week 2]

ရူပဗေဒ ပညာ ရှင် တယောက်ဖြစ်တဲ့ Richard Feynmanက လေ့လာတဲ့ နေရာမှာ သုံးတဲ့ နည်း စနစ်တစ်ခု ကို ပြောဘူးတယ်။ feynman technique ပေါ့။ သူပြောတာက ကိုယ် လေ့လာနေတဲ့ အကြောင်းအရာကို စာ ရွက် ပေါ်မှာ ကိုယ့်ကိုကိုယ် အရင် ရှင်းပြခိုင်းတယ်။ အဲ့မှာ မရှင်း လင်းတဲ့ အချက် တွေ တွေ့ လာတဲ့ အခါ မှာ တခါ စာပြန်ဖတ်ပြီးတော့ ကိုယ် လိုတဲ့ အပိုင်းကို ထပ်ရှင်းပြ ။ ဒီလိုနဲ့ပဲ ဒီ ဘာသာ ရပ် အပေါ်မှာ နားလည်မှု ပိုလာမယ် ။ ပိုကျွမ်းကျင်လာမယ်တဲ့။

အခု ကျနော် week တစ်ခု ချင်း စီ အလိုက် ရေးတာလည်း ပိုနားလည်လာမလား ဆိုပြီးတော့ ရေးကြည့်တာပါ။ ခက် တာက တခါတလေကြတော့ quizz တွေသာ အောင်သွားတယ်။ အခု ထိရွာလည်နေတော့ ဘယ်လို ရှင်းလို့ ရှင်းရမှန်းတောင် မသိတော့ဘူး။ နားလည်သလောက်ပဲ ရေးကြည့်မယ်။ အမှားကတော့ ပါမှာပါပဲ။

Chunking ဆိုတာ

Week 2 မှာ chunking အကြောင်းကို သင်ရတယ်။ chunk ဆိုတာ ကျနော် သိတာက tiny bit of information လို့ သိထားတယ်။ chunking ဆိုတာ chunk လေး တွေအဖြစ်အပိုင်းအပိုင်း ခွဲပြစ်တာ ။

ဒါပေမယ့် အခု သူဆိုလိုတဲ့ သဘောက chunk လေးတွေကို အဓိပ္ပါယ် ပြည့်ဝတဲ့ အစိတ်အပိုင်း တစ်ခု အဖြစ် စုစည်းတဲ့ သဘောဖြစ်နေတယ်။

အဓိပ္ပါယ် မပြည်စုံတဲ့ chunk တွေကို မှတ်ထားလို့လည်း ရပါတယ်။ ဒါပေမယ့် မှတ်ဉာဏ် ထဲ သာ ရောက်သွားတယ် သေချာ နားမလည်တော့ အချိန်ကြာတာနဲ့ အလိုလို မေ့သွားမှာပါပဲ။

chunk ဘယ်လို ဖြစ်လာလဲ ဆိုတာကို သူက စကား သင်ခါစ အရွယ်မှာ “အမေ” ဘယ်လို ခေါ်တက်လာတာလဲ ဆိုတဲ့ ဥပမာ တခု နဲ့ပေးထားပါတယ်။ အမေ လို့ ဆိုတာနဲ့ ကလေး စိတ်ထဲမှာ သူ့အမေ ပုံပန်းသဏ္ဍာန် နဲ့ တွဲမှတ်ထားပြီးတော့ “အမေ” ဆိုတဲ့ အသံရယ် ပုံရယ် နဲ့ တွဲပြီးတော့ အဓိပ္ပါယ် ပြည့်စုံတဲ့ chunk တစ်ခု ဖြစ်လာပါတယ်။

တဖြည်းဖြည်းနဲ့ ဒီ လို chunk လေးတွေပေါင်း လာလိုက်တာ နောက်ဆုံး စကားပြောတက်သွားသလိုပါပဲ။ အဲ့တော့ chunking ဆိုတာ chunk အသေးလေးတွေက နေတဖြည်းဖြည်းကြီး လာအောင် လုပ်တဲ့ process လို့ထင်တယ်။ တကယ် လို့ chunk သာဖြစ်သွားရင် အဓိက main idea ကို မှတ်ထားမယ် ဆိုတာနဲ့ နောက် detail process က အလို အလျောက် စဉ်းစားစရာ မလိုပဲ မှတ်မိပါတယ်။

သူဥပမာ ပြောထားတာက လူတွေအင်္ကျီဝတ် မယ် ဆိုရင် အင်္ကျီဝတ် မယ် ဆိုတော့ အတွေးတစ်ခု အစ ပြုလိုက်တာနဲ့ ကျန်တဲ့ detail process က အလိုအလျောက် လိုက်လာသလိုပါပဲတဲ့။

Chunking ဘယ်လို လုပ်မလဲ ?

လူ့ ဦး ဏှောက် မှာ working memory ရယ် long term memory ရယ် နှစ်ခု ရှိတဲ့ အကြောင်းကို ရှေ့ lecture မှာ ပြောထားတယ်။ working memory မှာ က slot လေးခု ပဲ ရှိတယ်။ အဲ့တော့ chunking လုပ်နေတဲ့ အချိန်မှာ တခြား distraction တွေဝင်လို့မရပါဘူး။ လေးခု စလုံး ကို အသုံးချပြီးတော့ အလုပ်လုပ်ရပါမယ်။ အဲ့တော့ ပထမ တချက် က

  • အာရုံစူးစိုက်မှု (focused attention)

နောက်တခုက သေသေချာချာ နားလည် ရပါမယ်။ သေသေချာချာ နားမလည်ပဲ နဲ့ ဖြစ်လာတဲ့ chunk တစ်ခုဟာ အခြား chunk တွေနဲ့ ချိတ်ဆက်ပြီးတော့ ပိုပြီးတော့ ကြီးမားလာ လာစရာ အကြောင်းမရှိပါဘူး။ ဒုတိယ အချက်က

  • နားလည် သဘောပေါက်မှု(understanding)

နောက် တတိယ အချက်ကတော့

  • လေ့ကျင့်မှု (practice)

ဖြစ်ပါတယ်။

chunking ဆိုတော့ chunk တွေ တဖြည်းဖြည်း ချင်း ကြီး လာအောင်လုပ်တဲ့ process ဖြစ်တာကြောင့် chunk အချင်းချင်း ဘယ်လို စပ်ဆက်ပေးရမလဲ ဆိုတာကို သိဖို့ လိုအပ်လာပါတယ်။ ဒီ အတွက် နောက် learning process တခု အကြောင်းကို မိတ်ဆက်ထားပါတယ်။

Top down learning process ဖြစ်ပါတယ်။

သူ အကြံပြုထားတာက အကြောင်းအရာ တခုကို နားလည်သဘောပေါက်အောင် မကြိုးစားရသေးခင် ပထမဦးဆုံး ဒီအကြောင်းအရာဟာ ဘာတွေ နဲ့ ဖွဲ့စည်းထားလဲ ဆိုတာကို အရင်ကြည့်ခိုင်းတာပါပဲ။

Chapter တစ်ခု မဖြတ်ခင်မှာ အဲ့ chapter မှာ ပါတဲ့ article တွေ figure တွေ လိုက်ပြီးတော့ အရင် အကြမ်းဖတ် ထားရပါမယ်။ chunk တွေအချင်းချင်း ဆက်နွယ် မှုကို သိပြီးသားဖြစ်သွားတဲ့ အခါမှာ တခု လုံး နားလည်း ဖို့ အတွက် အထောက်အပံ ဖြစ်ပါတယ်။

လက်ရှိ လေ့လာနေဆဲ ဖြစ်တဲ့ အကြောင်းအရာ တစ်ခုကို မှတ်မိချင်တယ် ဆိုရင် များသောအားဖြင့်ကတော့ နောက် တခေါက်ပြန်ဖတ်လိုက်တာပါပဲ။ ဒါပေမယ့်ပြန်ဖတ်တာထက် စာရင် မှတ်မိမမှတ်မိ recall ပြန် ခေါ်ကြည့်တာ ဟာ ပိုပြီးတော့ ထိရောက်ပါတယ် တဲ့။ အဲ့မှာ မမှတ်မိတာတွေလည်း ထွက်လာမှာပဲ။ အမှား အယွင်း တွေလည်း ထွက်လာမှာပဲ ။ ဒါပေမယ့် ဒီ mistake တွေကို လက်ခံရပါမယ်တဲ့ recall ခေါ်ရင် ခေါ်ရင်း နဲ့ နက်နက်နဲနဲ ပိုပြီးတော့ နားလည်လာပါလိမ့်မယ် တဲ့။

Motivation

စိတ်အား တက်ကြွနေတဲ့ အခါ မှာ ဘယ် အရာ မလို လေ့လာရ လွယ်ကူနေတယ် ဆိုတာ ငြင်းချက် ထုတ်စရာ မရှိပါဘူး။ Dopamine တွေ ဟာ motivation ကို ဖြစ်စေပါတယ် တဲ့။ ဘယ်အချိန်မှာ dopamine တွေ ထွက်လဲ ဆိုတော့ ကိုယ် လုပ်ရပ် တစ်ခု ဟာ နောက် တချိန်မှာ ကိုယ့် အတွက် အကျိုးရလဒ် တစ်ခု ဖြစ်မယ် ဆိုတာကို ယုံကြည်နေတဲ့ အချိန်မှာ motivation ဖြစ်လာပါတယ်တဲ့။ အဲ့တော့ ​motivation ရချင်တယ် ဆိုရင် နောက် တချိန်မှာ ဒီ အလုပ် ကြောင့် ရမယ့် အကျိုးရလဒ်ကို ခဏတိုင်း အောက့် မေ့ရမယ် ပေါ့။ :P

နောက် သူပြောသွားတာတွေကတော့ သေသေချာချာ နားလည်ပြီးသားကို ထပ်ကာထပ်ကာ learn မလုပ်ဖို့ [overlearning]။ ပိုပြီးတော့ ခက်ခဲတဲ့ အပိုင်း ကို လေ့ကျင့်ဖို့ရယ်။

week 2 မှာ သူဘာပြောတယ် ဆိုတာ လောက်ပဲ နားလည်တယ်။ ဘယ်လို အသုံးချရမယ် ဆိုတဲ့ အဆင့်ထိ နားမလည်သေးဘူး။ ခေါင်းထဲ ရောက်လာတာ ဆိုလို့ top down learning approach ပဲ ရှိတယ်။ မျက်စိ မမြင် ရတဲ့ ပုဏ္ဏား ခြောက် ယောက် ဆင်ကြီးကို စမ်းကြည့်ပြီးတော့ ထင်ရာ ပြောနေတဲ့ အတိုင်းပဲ။ :D

Learning How to Learn [Week 3]

Procrastination

ဗမာ လို ဆိုရင်တော့ အချိန် ဆွဲတာ ၊ အလုပ် တခုကို နေ့ရွေ့ ညရွေ့လုပ်တာ ပေါ့။ တခါတလေ ဆိုရင် လုပ်နေရင်တန်းလန်း နဲ့ တောင် အချိန်ရအောင် ဆွဲသေးတယ်။ ဒီ ပြဿနာ က သင်ယူတဲ့ နေရာမှာ လုံးဝ အနှောက် အယှက်ဖြစ်တယ်။ ဒါပေမယ့် စာသင် ရတယ် ဆိုတာ ပျော်တယ် ဆိုတဲ့ ကောင်မှ မရှိပဲ။ ကိုယ် ဘယ် လောက် စိတ်ဝင်စားတယ် ဆိုစေဦးတော့ စစခြင်းတော့ စာကြည့်ရအောင် ဆိုတာ နဲ့ တီ ကို ဆား နဲ့ တို့ လိုက်သလိုပဲ။ အဲ့ လို ဖြစ်တာ ကလည်း ကိုယ် တယောက် ထဲ ဖြစ်နေတာ မဟုတ်ပါဘူးတဲ့။ ဒါဟာ လူတိုင်း ဖြစ် နေကြ ဖြစ်ရိုး ဖြစ် စဉ် ကိစ္စဖြစ် တဲ့ အတွက် ဘယ်လို စီမံ မယ် ဆိုတဲ့ ကိစ္စကပဲ အရေးကြီးပါတယ်တဲ့။

ရှေ့ lecture တွေမှာ ပြောခဲ့ တဲ့ အထဲမှာ ၂၅ မိနစ် လုပ် ၅ မိနစ် နား ဆိုတဲ့ pomodoro နည်း စနစ် အကြောင်းကို ပြောပြထားတယ်။ ဘယ်သူမဆို 25 မိနစ် လောက်တော့ အာရုံ ကောင်းကောင်း စိုက် နိုင်မှာပဲပေါ့။ ဒါပေမယ့် ခက်တာက နည်းစနစ် ဆိုတာ က လူကလုပ်တာ ဆိုတော့ လူလုပ်တဲ့ စည်းကမ်း ကို လူက ဖောက်ဖို့ ရာ ဘာခက်တယ် မှတ်လို့။ လုပ်ရကိုင်ရတာ ပျင်း လာရင် အချိန် ဆွဲဖို့ အကြောင်းကို အလို အလျောက် ရှာ တော့တာပါပဲ။ ဒီ အချိန်မှာ ပုရွက်ဆိတ် လမ်း လျှောက်တာက အစ စိတ် ဝင်းစားစရာ ကောင်းနေမှာပဲ ။ အဲ့တော့ မရရအောင် ရှောင်ထွက်ပြီးတော့ နေ့ရွေ့ညရွေ့လုပ်ပါတယ်။

Procrastination ဖြစ်ပုံ အဆင့်ဆင့်

learning လုပ် ရတာ ပျင်းလာတယ် ဆိုရင် ရှောင်ထွက် ဖို့ အကြောင်းရှာပါတယ်။

ဥပမာ — fb က noti တက်နေပြီ။ ငါတင်ထားတဲ့ post အခြေ အနေလေး သွားကြည့်ဦးမှပါ။ သွားကြည့်လိုက်ရင် သေချာတယ် အနည်း ဆုံး နာရီဝက်ပဲ ။ noti တက်တာကို တုံပြန် ပုံ ပေါ် မူတည်ပြီးတော့ procrastinate ဖြစ်မဖြစ်ကွဲသွားပါပြီ။ ဒီ အလေ့အကျင် စွဲသွားရင် အမြဲတမ်း procrastinate ဖြစ် နေမှာပါပဲ။ သူကတော့ အလေ့အကျင့် တစ်ခု ဖြစ် ဖို့ အတွက် အချက် လေးချက် နဲ့ ရှင်းပြထားပါတယ်။

  1. The cue
  2. The routine
  3. The reward
  4. The belief

The cue ဆိုတာ အစပြုတဲ့ distraction signal ပေါ့။ ဥပမာ text message ဝင်လာလို့ ဖုန်း ကိုင် လိုက်တယ်။ ဖုန်းမှ ကိုင်မိတဲ့ နောက်တော့ သီချင်း လေး နားထောင် ဦးမှ fb သုံး ဦးမှ အစရှိသဖြင့် လုပ်ငန်း စဉ်တွေ ထပ်လာပါတယ်။ အဲ့တာကိုတော့ routine လို့ ခေါ်ပါတယ်။ သီချင်း နားထောင်တာ facebook သုံးတာ ဟာ learning လုပ်တာထက်တော့ ကိုယ့် အတွက် ပိုပြီးတော့ သက်တောင့်သက်တာ ဖြစ် စေမှာကတော့ သေချာပါတယ်။ အဲ့ဒီ သက်တောင့်သက်တာ ဖြစ်စေတဲ့ အကျိုးရလဒ် ကို တော့ reward လို့ ခေါ်ပါတယ်။ belief ဆိုတာက ဒီအလေ့အကျင့်အပေါ် စွဲမြဲနေတာ ဒီလို လုပ်ခြင်းအားဖြစ် ပိုပြီးတော့ သက်တောင့်သက်တာဖြစ် စေတယ် ဆိုတာကို ယုံနေတာပါ။ အမှန်တကယ်ကတော့ procrstinate ဖြစ်သွားရင် ပြီးသင့်တဲ့ အလုပ်က အချိန်မှီမပြီးတော့ deadline နီးရင် ခွေးလိုနွားလို ရုန်းရတော့မယ် ဆိုရင် ပိုဆိုးတာကို မမြင်တော့ဘူး။ ဒီလို နဲ့ သံသရာ လည်တော့တာပါပဲ။

ဒီ လေးချက်ကို နားလည်ရင် ဒီအကျင့်ကို ဖျက် ဖို့ လွယ်ပါတယ်တဲ့။

  • procrastinate ဖြစ်စေတဲ့ distraction တွေ ဝေးတဲ့ ပတ်ဝန်းကျင် တခု ဖန်တီးရပါမယ်

စာကြည့်တော့မယ် ဆိုရင် fb logout ထားတာတို့။ ဆိတ်ငြိမ်တဲ့ နေရာသွားကြည့်တာတို့ ပေါ့။

ဒီကြားတဲ့ ဝင် လာရင် ပထမဆုံး cue စဖြစ် တဲ့ အချိန်ကို သတိ ကပ်ပြီးတော့ ထိမ်းထားနိုင်မယ် ဆိုရင် procrastinate မဖြစ်တော့ပါဘူးတဲ့။

  • ကောင်းတဲ့ အလေ့အကျင့် တစ်ခု အတွက် ကိုယ့် ကိုယ်ကို reward လုပ်ပေးသင့်ပါတယ်။

reward ကတော့ ရှိပြီးသားပေါ့ လေ ။ အလေ့အကျင့်ကောင်းတခု ကြောင့် အနည်း ဆုံးတော့ procrastinate မဖြစ်ဘူးပေါ့။ ဒါပေမယ့် ကိုယ့် ကို ကိုယ် ဆုချ တော့ motivation ပိုတိုးတာပေါ့။ ရှေ့ week မှာလည်း ဒီ အကြောင်းပါပါတယ်။ Dopamine တွေတက်လာအောင် reward ဆိုတာ ရှိသင့်ပါတယ်။

Process Vs Product

ခုဏ တုန်း က procrastinate ဖြစ်တာ ဟာ ဝင်လာတဲ့ distraction ကို ကောင်းကောင်းမတုံပြန် နိုင် တဲ့ အတွက် ဖြစ်တယ် လို့ ပြောခဲ့ပါတယ်။ ဒါပေမယ့် ကိုယ် ကသာ learning လုပ်ရတာ ပျော်နေပြီ ဆိုရင် ဘယ် လောက် distraction ဝင် ဝင် သူ့ အလုပ်ကိုတော့ လုံးဝ အနှောက် အယှက် ဖြစ်တော့မယ် မထင်ပါဘူး။ အဲ့တော့ ဘာကြောင့် သက်တောင့်သက်တာမဖြစ်တဲ့ လဲ ဆိုတဲ့ မေးခွန်း ရှိလာပါတယ်။

မိုင်တစ်ထောင် ရဲ့ ခရီး အစ က ပထမ ခြေလှမ်း ပါ။ ခြေလှမ်း တလှမ်း ထဲ နဲ့ ခရီး အဆုံး ရောက်သွားမှာမလုပ်ပါဘူး။ တလှမ်း ချင်း လှမ်း နေ တဲ့ အချိန်မှာ မိုင်တစ်ထောင် မပြည့်သေးဘူးလား မပြည့်သေးဘူးလား လို့ တွေးပူးနေ ခြင်းဟာ စိတ်ပျက် လက်ပျက် ဖြစ် စေယုံကလွဲပြီးတော့ ဘာမှ ဖြစ်လာမှာ မဟုတ်ပါဘူး။

အဲ့တော့ ကျနော် တို့ အလုပ်လုပ်နေ တဲ့ အချိန်မှာ process တစ်ခုခြင်း ဆီ အပေါ်မှာပဲ အာရုံ စိုက်ရပါမယ်။ process ကောင်း ရင် product ကောင်းမှာပါပဲ။

Juggling life and learning

ကျောင်း သားတွေ ဆိုတာ တစ်ဘာသာ ထဲ့ သင်ရတာ မဟုတ်ပဲ။ သူတို့ မှာ learning လုပ် ဖို့ရာ အလုပ် တွေအများကြီးပဲ။ ဟိုဘာသာလည်း ကြည့်ချင် ဒီ ဘာသာလည်း ကြည့်ချင် ဦးဏှောက် ထဲမှာ task list တွေ အများကြီး မထားသင့်ပါဘူး။

ဘာလို့ လဲ ဆိုတော့ week 1 မှာ ပြောခဲ့ သလိုပဲ လူ့ ဦးဏှောက်မှာ working memory အတွက် slot လေးခုပဲ ရှိတာမှာ task list တွေနဲ့ ပြည့်နေရင် အာရုံ စိုက်တဲ့ နေရာမှာ အနှောင်အယှက် သွားဖြစ်စေပါတယ့် အဲ့ အတွက် daily task list ကို ထားရပါမယ်။

ည အိပ်ယာ မဝင်ခင်မှာ မနက် ဖြန် လုပ်မယ့် အလုပ်ကို စာရင်း လုပ် ထားပေးရပါမယ်။

နောက် တခုက စာကြည့်မယ် ကွာ ကြည့် နိုင် သလောက် ကြည့်မယ် ပြီး မှ နားမယ် ဆိုတာ မျိုး မလုပ်ရပါဘူး။ ဘယ်အချိန် နားမယ် ဆိုတာကို တိတိကျကျ သက်မှတ်ပေးရပါတယ်။ တစ် ခါ တည်း ပုံ အောကြည့်မယ့် အစား အချိန်နည်းနည်း အကြိမ်များများ ကြည့်သင့်ပါတယ်(space repeatation)။

Memory

လူ့ ဦး ဏှောက် က အမြင် နဲ့ ပတ်သက် ရင် ပိုပြီးတော့ မှတ်မိလွယ် တဲ့ သဘော ရှိတယ်။ ဥပမာ — ကိုယ် တခါမှာ မရောက်ဘူးတဲ့ အိမ် ကို အလည် သွားတယ် ဆိုပါတော့။ အနည်း ဆုံး ဒီ အိမ်ဧည့် ခန်း ရဲ့ ဖွဲ့စည်းပုံကို ရက် အတန်ကြာ အောင် မှတ်မိ နေသလိုပဲ။

အဲ့တော့ မှတ်သားစရာလေးတွေကို ပုံ နဲ့ တွဲ မှတ်တာ ဟာ ပိုကောင်းတယ်။

ဥပမာ w = mg ဆိုတဲ့ equation ဆိုပါတော့ ။

အရာ ဝတ္ထုတွေ ရဲ့ အလေး ချိန် ဆိုတာ ဒီ အရာ ဝတ္ထု ရဲ့ ဒြပ်ထု (mass) အပေါ်မှာ သက် ရောက် နေတဲ့ Gravity (ဒြပ်ဆွဲအား) ဖြစ်ပါတယ်။

အဲ့တာ ကို ကျနော် တို့ ကို့ weight မတဲ့ မောင် (mg) လို့ ဆရာ ကမှတ်ခိုင်းတယ်။ဘယ်လောက် မှတ်မိလဲ ဆိုတော့ အခု ထိတောင် မမေ့ သေးဘူး။ equation လွယ်လို့လည်း ဖြစ်မှာပေါ့လို့ ယူဆ လို့ရတယ်။ ဒါပေမယ့် လူ့ဦးဏှောက် ဟာ အမြင် နဲ့ ပတ်သက် ရင်ပိုမှတ်မိလွယ်တာမို့ တွဲမှတ်တာဟာ ပိုကောင်းပါတယ်။ (ဒီ နည်း စနစ်တော့ကျနော်လည်း မသုံးဘူးသေးဘူး)

နောက် ပြီးတော့ မှတ်ရခက်တဲ့ စာရင်းတွေကို အတိုကောက် မှတ်ရပါတယ်။

ဥပမာ — ရသ ကိုးပါးကို

ချစ်၊ ရွှင်၊ သနား၊ တည်ကြား၊ ကြမ်းကြုပ်၊ စပ်ဆုပ်၊ ကြောက့်ရွှံ၊ ရဲဝံ၊ အံဩ လို့ မှတ်သလိုမျိုးပေါ့။

နောက် တခု က စိတ်ကူး နန်းတော် ဆောက် ခိုင်းပြီးတော့ ကိုယ် မှတ်မိ စေချင်တာ လေးတွေသွား ထားခိုင်းတာ Sherlock Holmes ရဲ့ mind palace လိုပေါ့။

အဲ့ နည်း စနစ် ကို တော့ တခါမှတော့ မစမ်းကြည့်ရသေးတော့ ကောင်းတယ် မကောင်းဘူးတော့ မပြောပါဘူး။ ဒါပေမယ့် တကယ် အလေ့အကျင့်ရရင်တော့ အကျိုးတော့ ရှိမှာပါပဲ။

နိဂုံးချုပ်

ဒီ အခန်း က procrastination ကို handle လုပ် ဖို့ အတွက် တော့ တကယ် အထောက် အကူဖြစ်ပါတယ်။ သူပြောသလို့ daily task ကို တော့ plan မချဖြစ်ပေမယ့် ။ process ကိုပဲ အာရုံစိုက်တဲ့ အခါမှာ အတော် စိတ်သက် သာရာ ရလာတယ်။ ဒါပေမယ့် reward ဆိုတာကလည်း အရေးကြီးတယ်။ ကိုယ် learning လုပ် တိုင်းမှာ ငါလုပ် နေတာ နောက် ကျရင် အကျိုး ရှိလာမှာ ဆိုတာကို ခဏ ခဏ နှလုံးသွင်း မပေးရင် motivation တက်မလာ၊ လုပ်ချင်ကိုင်ချင်စိတ် ပျောက်သွားတာတွေ ဖြစ်ကုန်ရော။

စာရေးဆရာ

စာရေး တဲ့ နေရာမှာ စကားပြေ နဲ့ရေးတာ ရယ် ၊ စကားပြောနဲ့ ရေး တာ ရယ်မှာ ​စကားပြောနဲ့ ရေးတာ က ရေးရလွယ်တယ်။​ ဖတ် ရပေါ့တယ် လို စိတ် ထဲ ထင်တယ်။ အဲ့လို လည်း မဟုတ်ပြန်ဘူး ။​ စကားပြောနဲ့ ရေး တဲ့ စာတွေ ဥပမာ လူထု ဒေါ်အမာ ရေးတဲ့ အမေ့ ရှေးစကား ကို ဖတ်ရ ပေါ့တယ် လို့ မှ ပြောလို့ မရတာ ။
